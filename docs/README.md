# OctoAcme Project Management Docs

This folder is the onboarding entry point for OctoAcme's project management process. OctoAcme runs projects through a lightweight, iterative lifecycle: teams align on the problem and success metrics during initiation, turn approved work into a prioritized backlog and release plan during planning, deliver in small increments with regular demos and status tracking during execution, manage risks and stakeholder communication throughout delivery, and close the loop with controlled releases, retrospectives, and follow-up improvements. Clear roles across project management, product management, and engineering keep ownership, decisions, and communication aligned from kickoff through continuous improvement.

## Recommended reading path

- Start with the overview for the core principles, roles, and artifacts.
- Use initiation and planning to define goals, scope, milestones, and dependencies.
- Follow execution, risk, and release guidance while work is in progress.
- Use retrospectives and the roles guide to reinforce accountability and continuous improvement.
